# FastAPI Template

This repository is a starter template for FastAPI projects. It includes:

- local development with `uv`, `just`, and Postgres 17 through Docker Compose
- SQLAlchemy Core with an async engine, pessimistic connection pooling, and Alembic migrations
- request-scoped database transactions through FastAPI dependencies
- a feature-slice example under `src/items`
- global Pydantic models with explicit timezone handling during JSON export
- global typed HTTP exceptions
- production-oriented Docker and Gunicorn configuration
- JSON logging for production
- Sentry setup for deployed environments
- middleware guidance: prefer pure ASGI middleware over `BaseHTTPMiddleware` ([details](docs/middleware.md))

## Local Development

### Install just

macOS:

```shell
brew install just
```

Debian/Ubuntu:

```shell
apt install just
```

Other installation options: https://github.com/casey/just

### Install uv

```shell
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Other installation options: https://docs.astral.sh/uv/getting-started/installation/

### Setup

```shell
cp .env.example .env
uv sync
just up
just migrate
```

`just up` starts Postgres. `just migrate` upgrades the database to the latest
Alembic migration.

### Run the API

```shell
just run
```

With an explicit logging config:

```shell
just run --log-config logging.ini
```

The API exposes a healthcheck at `/health`.

## Commands

```shell
just lint                 # ruff format + ruff check --fix
just mm <migration_name>  # create an autogenerated Alembic migration
just migrate              # upgrade database to head
just downgrade -1         # downgrade one migration
just ps                   # show Docker Compose services
just down                 # stop and remove Docker Compose services
```

Migration files use the `YYYY-MM-DD_slug` naming format.

## Application Structure

Application code lives in `src/`.

- `src/main.py` creates the FastAPI app, configures CORS, registers exception handlers, and includes routers.
- `src/config.py` loads settings from `.env` with `pydantic-settings`.
- `src/database.py` creates the async SQLAlchemy engine and exposes `get_db_connection()`.
- `src/items` is an example feature slice with router, service, dependencies, schemas, models, exceptions, and constants.

The database layer uses SQLAlchemy Core `Table` objects attached to shared
metadata. Routes receive an `AsyncConnection` through FastAPI dependencies, and
services execute SQL without committing manually. Commit and rollback are handled
by the request-scoped transaction in `get_db_connection()`.

## Deployment

Production deployment uses Docker and Gunicorn:

```shell
docker compose -f docker-compose.prod.yml up -d --build
```

The production image runs as a non-root user, uses Gunicorn with dynamic worker
configuration, emits JSON logs, and defines a `/health` based container
healthcheck.
